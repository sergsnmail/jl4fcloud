1. Упрощен пайп-лайн на клиенте и сервере. JSON конвертируется сразу в байты, минуя строку. В связи с этим удалены кодеки String. Также изменены MessageEncoder и MessageDecoder.
2. В пайп-лайн в начало включены LengthFieldBasedFrameDecoder и LengthFieldPrepender.
3. В код проекта вставлены поясняющие комментарии, а также удалены мусорные строки.
4. Исправленые небольшие неточности по работе клиента с сервером.
5. Полностью переработан механизм передачи файлов на сервер от клиента. Предыдущий вариант удален.
6. Немного доработано окно клиента для корректной обработки ресайза окна.

Текущие итоги:
Клиент
 - Работа клиента осуществляется только по направлению к серверу. Т.е клиент корректно отправляет файлы из каталога синхронизации на сервер.
 - Корректкно реагирует на добавление каталогов в папке синхронизации.
 - Корректно отрабатывает при массовом копировании файлов в каталог синхронизации.
 - Корректно передает на сервер файлы большого размера (проверялось на файлах 7-8ГБ).
 - Корректно осуществляет индикацию передачи файлов в интерфейсе пользователя (прогресс-бар, количество отправленных пакетов/ общее количество пакетов, имя отправляемого файла).
 - Отправляет данные для регистрации нового пользователя.
Сервер
 - Регистрирует нового пользователя
 - Предоставляет информацию по запрошенному файлу
 - Отправляет список файлов пользователя клиенту
 - Оптимально организует хранение файлов в каталоге операционной системы.
 - Принимает файлы от клиента и регистрирует в БД
